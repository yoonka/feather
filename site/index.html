WHAT.IS.THIS FRAME 02 / 06

NOT ANOTHER POSTFIX

Most mail servers are monoliths with a thousand config knobs and a 1990s threat model. Feather is the opposite shape — a thin, opinionated runtime that calls into your code for every decision: who can authenticate, where a message routes, how it's stored, what gets quarantined.

The same engine runs as a submission agent on :587, a transfer agent on :25, or a delivery agent on a private port — different pipelines, same primitives. The wire protocol is RFC; everything above is yours.

Written in Elixir. Built on the BEAM. Tested under fire. The kind of mail server you'd ship if you were building it for the next forty years instead of recovering from the last.

First-class auth-results adapters. Each check writes a structured entry to meta[:auth_results]; a transformer renders one RFC 7601 Authentication-Results header on the delivered message.

// how it looks

just a list

A Feather pipeline is an Elixir list of {Adapter, opts} tuples. Order matters — adapters run top-to-bottom, top to bottom. Any one of them can stop the message and write a 5xx code with the reason.

No DSL. No YAML. No magic strings. Just code — which means you can read it, test it, and version it like the rest of your service.

The example on the right is the actual inbound MTA pipeline used on mta.maxlabmobile.com — logger, backscatter guard, SPF/DKIM/DMARC, then forward to the MDA.

pipeline.exs elixir · 31L 
# /etc/feather/pipeline.exs — INBOUND :25
[
  {FeatherAdapters.Logging.MailLogger,
   backends: [:console], level: :info},

  {FeatherAdapters.Access.BackscatterGuard,
   mode: :strict,
   guards: [FeatherAdapters.Access.BackscatterGuard.RcptDomain]},

  # RFC 7601 — record auth result, never block.
  {FeatherAdapters.AuthResults.SPF,   on_fail: :pass_through},
  {FeatherAdapters.AuthResults.DKIM,  on_fail: :pass_through},
  {FeatherAdapters.AuthResults.DMARC, on_fail: :pass_through},

  # Hand off to the local MDA.
  {FeatherAdapters.Delivery.SMTPForward,
   server: "127.0.0.1", port: 2528,
   tls: :never,
   transformers: [
     {FeatherAdapters.Transformers.AuthenticationResults,
      authserv_id: "mta.maxlabmobile.com"}
   ]}
]
